ChatGPT 连接本地只读 MCP 网关教程

把 Windows 本机的 local-project-gateway 暴露成只读 MCP 工具，让 ChatGPT 能通过自定义 MCP 应用读取白名单项目的文件树和文件内容。

Windows PowerShell Node.js MCP Streamable HTTP Caddy HTTPS 反代 ChatGPT Developer Mode Readonly Only

目标

最终让 ChatGPT 通过下面的地址连接你的本地 MCP 服务：

https://mcp.hysons.top/mcp

本地 MCP 服务仍然运行在：

http://127.0.0.1:3334/mcp
本教程只做只读能力：list_projects、list_tree、read_file。不要开放 write_file、delete_file、run_command、git push 这类能力。

架构

ChatGPT 自定义 MCP 应用
https://mcp.hysons.top/mcp
Caddy HTTPS 反向代理
127.0.0.1:3334 / MCP Streamable HTTP server
127.0.0.1:3333 / readonly local-project-gateway
projects.json 白名单项目

关键点：ChatGPT 在云端运行，所以它不能访问你电脑上的 127.0.0.1。必须使用公网可访问的 HTTPS 域名。

3. 创建 start-gateway.ps1

$ErrorActionPreference = "Stop"

Set-Location $PSScriptRoot

Write-Host "Starting readonly HTTP gateway..."
Write-Host "Project: $PSScriptRoot"
Write-Host "URL: http://127.0.0.1:3333"

npm start

4. 创建 start-mcp-http.ps1

这一版兼容直接运行脚本，也能避免 $PSScriptRoot 为空导致 Join-Path 报错。

$ErrorActionPreference = "Stop"

$ProjectRoot = if ($PSScriptRoot) {
    $PSScriptRoot
} else {
    (Get-Location).Path
}

Set-Location $ProjectRoot

$envFile = Join-Path $ProjectRoot ".env.local"

if (!(Test-Path $envFile)) {
    throw ".env.local not found. Create it first and set MCP_GATEWAY_TOKEN."
}

Get-Content $envFile | ForEach-Object {
    $line = $_.Trim()

    if ($line -eq "" -or $line.StartsWith("#")) {
        return
    }

    $parts = $line -split "=", 2

    if ($parts.Count -ne 2) {
        throw "Invalid .env.local line: $line"
    }

    $key = $parts[0].Trim()
    $value = $parts[1].Trim().Trim('"').Trim("'")

    if ($key -eq "MCP_GATEWAY_TOKEN") {
        $env:MCP_GATEWAY_TOKEN = $value
    }
}

if ([string]::IsNullOrWhiteSpace($env:MCP_GATEWAY_TOKEN)) {
    throw "MCP_GATEWAY_TOKEN is empty or missing in .env.local"
}

Write-Host "Starting MCP HTTP server..."
Write-Host "Project: $ProjectRoot"
Write-Host "URL: http://127.0.0.1:3334/mcp"
Write-Host "Token: loaded from .env.local"

npm run mcp:http

5. 启动两个服务

窗口 A：

cd D:\local-project-gateway
.\start-gateway.ps1

窗口 B：

cd D:\local-project-gateway
.\start-mcp-http.ps1

6. 本地验证

curl.exe -i http://127.0.0.1:3334/health

预期：

HTTP/1.1 200 OK
{"ok":true,"service":"local-project-gateway-http-mcp","mode":"readonly","mcpPath":"/mcp"}

再跑 smoke：

$env:MCP_GATEWAY_TOKEN = (Get-Content .env.local | Select-String "MCP_GATEWAY_TOKEN").ToString().Split("=",2)[1]
npm run mcp:http:smoke

Stage 3.2：公网 HTTPS 与 Caddy

ChatGPT 不能连 http://127.0.0.1:3334/mcp，必须连公网 HTTPS：

https://mcp.hysons.top/mcp

方案 A：标准 Bearer Token 透传

适合支持自定义请求头或 Bearer token 的 MCP 客户端。

mcp.hysons.top {
    reverse_proxy 127.0.0.1:3334
}

方案 B：ChatGPT Developer Mode 先跑通用法

如果 ChatGPT 创建应用页只能选择“无身份验证”，可以让 Caddy 给后端注入 token。

mcp.hysons.top {
    reverse_proxy 127.0.0.1:3334 {
        header_up Authorization "Bearer REPLACE_WITH_YOUR_TOKEN"
    }
}
方案 B 只是为了先连通 demo。公网任何能访问该域名的人，都可能通过 Caddy 被自动加上 token。正式接真实项目之前，应改成真正的认证方案，例如 OAuth / MCP auth metadata，或者至少增加更严格的访问控制。

重载 Caddy

cd C:\Users\zengqu\Desktop\v2rayN-windows-64\caddy
caddy fmt --overwrite .\Caddyfile
caddy reload --config .\Caddyfile

如果 caddy 能运行，但 .\caddy.exe 不能运行，说明 Caddy 在 PATH 里，不在当前目录。直接用 caddy 即可。

公网健康检查

curl.exe -i https://mcp.hysons.top/health

预期：

HTTP/2 200
{"ok":true,"service":"local-project-gateway-http-mcp","mode":"readonly","mcpPath":"/mcp"}

ChatGPT 创建 MCP 应用

  1. 打开 ChatGPT 设置。
  2. 进入 应用 / Apps。
  3. 打开 开发人员模式 / Developer Mode。
  4. 返回应用页，点击 创建 / Create / Create custom app。
  5. 填写 MCP 服务器 URL。
字段填写内容
名称驱动本地 MCP
描述读取本地 demo 项目的只读 MCP 工具
MCP 服务器 URLhttps://mcp.hysons.top/mcp
身份验证无身份验证，不要选 OAuth
如果选择 OAuth，但你的 MCP server 没实现 OAuth discovery、authorization endpoint、token endpoint 等元数据，创建应用会失败。当前教程里的本地 MCP server 不是 OAuth server。

常见问题

1. $PSScriptRoot 是空的

原因：你把 .ps1 内容一行行粘进 PowerShell 运行了。$PSScriptRoot 只有在脚本文件中运行时才稳定。

正确做法：

notepad .\start-mcp-http.ps1
.\start-mcp-http.ps1

2. ChatGPT 创建应用时报 OAuth 错

典型错误：

获取 OAuth 配置时出错
502, message='Bad gateway', url='http://127.0.0.1:4750'

原因：你选了 OAuth，但当前 MCP server 不是 OAuth server。改成 无身份验证。

3. .\caddy.exe 不是可识别命令

说明当前目录没有 caddy.exe，但系统 PATH 里可能有 caddy。

where.exe caddy
Get-Command caddy
caddy reload --config .\Caddyfile

4. 本机访问公网域名失败，但外网能访问

这可能是 IPv6 / NAT loopback 问题。本机可以用：

curl.exe -k -i --connect-to mcp.hysons.top:443:127.0.0.1:443 "https://mcp.hysons.top/health"

5. curl: Failed to connect to mcp.hysons.top port 443

按顺序检查：

curl.exe -i http://127.0.0.1:3334/health
netstat -ano | findstr ":443"
nslookup mcp.hysons.top
curl.exe https://api.ipify.org

重点看：MCP 本机是否活着、Caddy 是否监听 443、DNS 是否指向当前公网 IP、路由器和 Windows 防火墙是否放行。

安全边界

  • 不要提交 .env.local。
  • 不要把 MCP token、Cloudflare token、SSH key 发给 ChatGPT 或提交 GitHub。
  • 不要开放 C:\、D:\、C:\Users\zengqu 这类大目录。
  • 只通过 projects.json 白名单开放 demo 或明确允许的项目。
  • 公网接 ChatGPT 时，先只挂 demo。真实项目要等认证方案收紧后再接。
  • 继续禁止 write_file、delete_file、run_command、远程 git commit、远程 git push。
